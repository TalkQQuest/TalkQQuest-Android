# 담당 분담 — 내 폴더와 API

내가 **어느 폴더를 만지고, 기능명세서 어느 부분을 보는지**만 정리한 문서입니다.
전체 흐름은 `작업순서.md`, 구조 이유는 `폴더 구조와 이름 규칙.md`, 화면 목록은 `NAVIGATION.md`, 규칙은 `CONVENTIONS.md` 를 보세요.

---

## 먼저 — "기능 ID(A101, H101 …)"가 뭔가요?

**기능명세서**는 앱 기능을 쭉 나열한 문서인데, 기능마다 **번호(ID)** 를 붙여놨습니다. 도서관 청구기호 같은 거예요.

```
알파벳 = 기능 묶음(카테고리)      숫자 = 그 안의 개별 기능
   H101  →  H(커뮤니티) 그룹의 101번 기능
```

알파벳 구분: `A 인증 · B 온보딩 · C 홈/미션 · D 대화 · E 피드백/리포트 · F 아카이브 · G 프로필 · H 커뮤니티 · I 모임 만들기 · J 리워드/알림`

즉 아래의 `H101~H102`는 **"기능명세서에서 커뮤니티 관련 101~102번 항목"** 이라는 뜻입니다. 이 번호 자체가 API가 아니라, **이 번호로 기능명세서를 찾아가면** 그 화면의 API·요청/응답 필드·에러코드 같은 **상세가 다 적혀 있습니다.**

> 📄 **자세한 내용은 "톡퀘스트 기능명세서(PDF)"를 직접 보세요.** 아래 번호로 검색하면 내 담당 부분으로 바로 갈 수 있습니다.

## A담당 (지니/전준호) — 진입 · 아카이브 · 프로필 (화면 17개)

**건드릴 폴더**
```
feature/auth/        (스플래시·로그인·회원가입·이메일 인증)
feature/onboarding/  (성향·상황·목표 선택)
feature/archive/     (아카이브 홈·검색·대화기록·저장문장)
feature/profile/     (프로필)
```
**기능명세서에서 찾아볼 번호**
- 인증 `A101~A103` · 온보딩 `B101~B103` · 아카이브 `F101~F103` · 프로필/설정 `G101~G103`

---

## B담당 (이도/윤기수) — 미션 · AI 대화 (화면 9개)

**건드릴 폴더**
```
feature/home/          (홈 대시보드)
feature/mission/       (미션 목록·상세·대화 준비)
feature/conversation/  (대화하기·완료·XP·AI 피드백)
```
**기능명세서에서 찾아볼 번호**
- 홈/미션 `C101~C103` · 대화 `D101~D103` · AI 피드백 `E101`
  - ※ 피드백 1차는 실제 AI 아님(mock/템플릿)

---

## C담당 (훈/김재훈) — 커뮤니티 · 성장 리포트 (화면 9개 + 팝업 4개)

**건드릴 폴더**
```
feature/community/  (모임 검색·상세·만들기·내 모임)
feature/report/     (성장 리포트 월간·주간)
```
**기능명세서에서 찾아볼 번호**
- 커뮤니티 `H101~H102`(예외 H-1) · 모임 만들기 `I101~I103` · 리워드/알림 `J101·J103` · 리포트 `E102`
- 팝업 4개(이탈·게시완료·탈퇴×2)도 전부 C담당
- 리포트 진입: **아카이브 → 리포트 탭 → 항목 클릭 → 월간 → 주간** (아카이브 화면은 A담당이니 진입 연결만 맞추기)
